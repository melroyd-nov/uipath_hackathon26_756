## Recommended Scope

**Recommendation:** Solution — Maestro Flow (orchestration) + Coded Agent (AI analysis/analytics/Q&A) + Maestro BPMN (follow-up approval lifecycle) + Coded App (dashboard UI), backed by Data Fabric and an Orchestrator Storage Bucket + Trigger.
**Delivery model:** Cloud — full product catalog available, nothing blocked by the Constraint Gate.
**Blocked by platform:** none.
**Why a Solution and not a single product:** the source process (`call-center-transcript-process.bpmn`) spans four separately-buildable surfaces — an orchestration pipeline, an AI/analytics backend, a human follow-up lifecycle, and an end-user dashboard — each requiring its own specialist skill and its own deployable artifact, unified into one `.uipx`.
**Why the follow-up lifecycle is its own project, not inline HITL in the Flow:** the `pending → approved/rejected → in_progress → completed` lifecycle is multi-actor and indefinite-duration (waits on human turnaround). Modeling it as gateways + inline HITL inside `call-center-orchestration` would work mechanically but would force the per-call AI pipeline to block on human turnaround time. Splitting it into its own process lets `call-center-orchestration` finish its automated work immediately and lets the follow-up process run independently. Per explicit instruction, this project is built as **Maestro BPMN** (not Case Management) for now — same separation, different authoring product.

## 1. Solution Overview

**Business context:** Acme operates an insurance contact centre. Every customer call produces a transcript that must be analysed for sentiment, intent, compliance, escalation risk, and follow-up actions, then surfaced to supervisors for approval/tracking and to the business via a live KPI dashboard and natural-language Q&A.

**Objective:** build a UiPath-native call center analytics platform maximizing platform product coverage:

| Capability | UiPath Product |
|---|---|
| Orchestration pipeline | **Maestro Flow** |
| AI analysis & analytics | **Coded Agent** (Python) using UiPath LLM Gateway |
| Supervisor dashboard (15+ pages) | **Coded App** |
| Data persistence | **Data Fabric** entities |
| Transcript file ingestion | **Orchestrator Storage Bucket** + **Time Trigger** |
| Follow-up approval lifecycle | **Maestro BPMN** project (`call-center-followup-case`) with inline HITL tasks |
| Packaging & deployment | **`.uipx` Solution** deployed to Orchestrator (Cloud) |

**Primary source document:** `Docs/call-center-transcript-process.bpmn` — a 3-lane (System / UiPath Generative AI Service / Supervisor), 25-node BPMN process already authored with UiPath terminology (Data Fabric, UiPath Generative AI, UiPath Apps, UiPath Agent, UiPath Orchestrator), confirming this is a native implementation rather than a greenfield design.
**Secondary source:** `Docs/PROJECT_SUMMARY.md` — existing documentation of the predecessor system, used to recover concrete field names, endpoint shapes, and business logic not visible in the BPMN's documentation blocks alone (e.g. exact KPI SQL formulas, the friction/marketing scoring weights, the follow-up dedup rule).

### In Scope

- Transcript ingestion from a storage bucket (replacing local `.txt` / `transcripts.zip` reads)
- 7-way parallel AI analysis per call: sentiment, intents (≤3), trigger words, compliance, follow-up items (≤3), resolution/escalation/preverified/repeat flags, call summary
- Data Fabric persistence of `CallRecord`, `CallFollowup`, `AiUsage`
- Follow-up seeding + supervisor approval lifecycle (pending → approved/rejected → in_progress → completed)
- 8-KPI dashboard with RAG benchmark indicators
- 6 analytics modules (sentiment, escalation, compliance, resolution, intents/marketing, friction/trigger-words)
- Grounded natural-language Q&A over live analytics
- AI usage counters (`calls_today`, `total_calls`)

### Out of Scope

- Telephony recording itself (`Task_RecordCall` is the call centre's existing telephony system — out of this Solution's build scope; we consume its output files)
- Historical data backfill (a one-time task flagged `[SME REVIEW]` in §4 — out of the BPMN's described scope)
- MAS regulatory reporting integration (mentioned as an *impact* of compliance failure in the BPMN documentation, not as a system requirement)

## 2. Cross-Project Data Flow

```

        │  Time Trigger (daily, or on-upload)
        ▼
┌─────────────────────────── Maestro Flow: call-center-orchestration ───────────────────────────┐
│                                                                                                  │
│  Ingest node ──► Parallel fork (7 branches) ──► Agent tool calls ──► Parallel join              │
│       │                                              │                                          │
│       │                                      call-center-ai-agent (Coded Agent)                 │
│       │                                      [sentiment, intents, trigger-words, compliance,    │
│       │                                       followups, resolution-flags, summary]             │
│       ▼                                                                                          │
│  Data Fabric write node ──► CallRecord entity                                                   │
│       ▼                                                                                          │
│  Seed-followups node ──► CallFollowup entities (source=ai_generated, status=pending)             │
│       ▼                                                                                          │
│  Conditional split: has follow-ups? ──Yes──► Start case (fire-and-forget, does not wait) ──┐      │
│       │No                                                                                  │      │
│       ▼◄─────────────────────────────────────────────────────────────────────────────────┘      │
│  Dashboard refresh node (reads Data Fabric, computes 8 KPIs)                                     │
│       ▼                                                                                          │
│  Parallel fork (6 analytics modules) ──► Agent tool calls ──► Parallel join                      │
│       │                                      call-center-ai-agent (Coded Agent)                  │
│       │                                  [sentiment/escalation/compliance/resolution/             │
│       │                                   intent+marketing/friction+trigger-word analytics]       │
│       ▼                                                                                          │
│  End (Q&A is not part of this Flow — see Coded App → Agent below)                                │
│                                                                                                    │
└──────────────────────────────────────┬─────────────────────────────────────────────────────────┘
                                         │ (fire-and-forget, from "Start case" above)
                                         ▼
                  ┌──────────────────────────────────────────────────────────┐
                  │   Maestro BPMN: call-center-followup-case                │
                  │   Task_SupervisorReview ──reject──► Task_RejectFollowup  │
                  │       │approve                          (terminal)      │
                  │       ▼                                                  │
                  │   Task_AssignDetails ──► Task_TrackProgress              │
                  │       ──complete──► Task_MarkCompleted (terminal)        │
                  └──────────────────────────────────────────────────────────┘
        ▲                                          ▲                          ▲
        │  reads Data Fabric directly             │  embeds this process's  │  calls Agent for
        │  for list/detail/dashboard views        │  HITL forms             │  AI Insights chat
┌───────┴──────────────────────────────────────────┴──────────────────────────┴───────────────────┐
│                          Coded App: call-center-dashboard (15+ pages)                            │
└────────────────────────────────────────────────────────────────────────────────────────────────┘
```

**Integration points:**
- Flow → Agent: Flow invokes `call-center-ai-agent` as a tool/connector node for every AI-bearing step (7 extraction tasks + 6 analytics modules = 13 tool calls) — never calls the LLM Gateway directly, and never calls the agent's 14th tool (`ask`).
- Flow → Data Fabric: direct connector nodes for `CallRecord` insert, `CallFollowup` insert, `AiUsage` upsert.
- Flow → Maestro BPMN: fire-and-forget `StartFollowupProcess` node — the Flow does not wait for the process to resolve, and does not implement any of the approve/assign/track/complete state machine itself (that lives in `call-center-followup-case`).
- BPMN process → Data Fabric: reads/writes `CallFollowup` directly on every task, and reads `CallRecord` for display context.
- Coded App → Data Fabric: direct reads for list/detail/dashboard pages (no need to round-trip through the Flow for read-only views).
- Coded App → BPMN process: embeds its HITL forms for the follow-up approve/assign/track/complete actions rather than reimplementing them.
- Coded App → Agent: direct call to `ask()` for the AI Insights chat page (`AIInsightsPage`) — the only caller of this tool; Q&A is a synchronous user-initiated action and was removed from the Flow entirely (no caller ever invoked the Flow's copy of this path — see `call-center-orchestration-sdd.md` v1.3).
