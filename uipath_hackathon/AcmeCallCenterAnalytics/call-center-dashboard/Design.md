# APStudio — Style Reference
> Deep-navy enterprise SaaS with trust-blue accents; compact, information-dense, AI-forward.

**Theme:** Light

APStudio uses a dark-indigo brand identity (`#0F1F4C` / `#150958`) layered over crisp white card surfaces and a near-white page background, creating a professional finance-tool feel without being sterile. Interactive elements lean on a confident mid-blue (`#1E5EAC` / `#3B82F6`) while semantic status signals follow strict color roles — green for approval, red for rejection, amber for risk, purple for fraud. Type is set in tight, slightly-condensed weights with heavy use of uppercase micro-labels and tabular numerals for financial data.

## Do's and Don'ts

### Do's
- Use `tabular-nums` and `[font-feature-settings:'tnum']` on all financial amounts, dates, and IDs.
- Use brand-tinted borders (`rgba(15,31,76,0.08)` to `rgba(30,94,172,0.14)`) instead of neutral gray on cards adjacent to branded content.
- Apply layered shadows: a soft base (`0 2px 12px`) + a tighter shadow (`0 1px 3px`) for realistic card depth.
- Use `uppercase tracking-wide` (0.06em+) on all micro-labels, meta text under 11px, and category badges.
- Keep risk flag badges (`font-black`, letter-spaced, rounded-full) distinct from status badges (`font-semibold`, rounded-md).
- Use `leading-[1.5]` – `leading-[1.6]` for AI-generated text blocks; they need more breathing room.
- Apply `truncate` on vendor names in list rows; the column width is fixed.

### Don'ts
- Don't use raw Tailwind green (`text-green-600`) for approval states — use `text-emerald-700` or `text-[#10B981]` to match the token.
- Don't use black (`#000000`) for text — use `#0F1F4C` (brand-dark) or `#1A1A2E` (content-primary).
- Don't use `shadow-md` or Tailwind's named shadows on cards — all card shadows are custom brand-tinted `rgba` shadows.
- Don't mix font families in a single component. Body/meta = Inter; Headings = Poppins; Invoice amounts = `font-mono`; Card vendor names = Outfit.
- Don't use `border-gray-200` on brand-adjacent surfaces — use `border-[rgba(15,31,76,0.08)]` to maintain the indigo tint.
- Don't place AI cards without the gradient background; flat white background removes the AI differentiation signal.
- Don't animate status badges or standard cards — animation is reserved for AI components and loading states.

## Elevation

APStudio uses a four-tier model. All shadows are brand-tinted (not neutral black):

| Tier | Shadow | Used On |
|------|--------|---------|
| Flush | none | In-page labels, dividers, zebra rows |
| Raised (1dp) | `0 1px 4px rgba(15,31,76,0.06)` | Subtle cards, pills |
| Elevated (2dp) | `0 2px 12px rgba(15,31,76,0.10), 0 1px 3px rgba(15,31,76,0.06)` | Invoice cards, panels |
| Lifted (hover, 4dp) | `0 8px 24px rgba(15,31,76,0.14), 0 2px 6px rgba(15,31,76,0.08)` | Hovered cards |
| Floating (dropdowns, 8dp) | `0 4px 16px rgba(15,31,76,0.10)` | Dropdowns, tooltips |
| Dialog (modals, 16dp) | `0 24px 64px rgba(0,0,0,0.20)` | Standard modals |
| Sheet (full-screen, 32dp) | `0 40px 100px rgba(0,0,0,0.28), 0 4px 20px rgba(0,0,0,0.10)` | Large PO modals |

## Imagery

- **Avatars:** Initials-based (`w-7 h-7 rounded-lg bg-[#1E5EAC] text-white text-xs font-bold`), no photo support seen in components.
- **Icons:** Lucide React — consistently 14px–18px, `stroke-width-1.5` or `stroke-width-2`, colored with `text-current` or explicit `text-[color]`.
- **AI orb:** Radial gradient blur overlay (`bg-radial-[circle_at_70%_50%,rgba(59,130,246,0.22),transparent_60%]`) layered behind AI card content.
- **Background mesh:** `MeshGradient` component — animated SVG/canvas with floating gradient orbs, used on login page and portal splash screens.
- **SVG dot grid:** 3×3 pixel dot pattern used as a texture behind AI card content.
- **Charts:** Recharts with custom color palette from `--color-chart-*` tokens.
- No raster images, no hero photos — UI is entirely vector/code-generated.
